# Honeytong Screen Specs

## 1. 문서 목적

이 문서는 Honeytong 프로젝트의 주요 화면별 UI/UX 기준을 정의한다.

Honeytong은 벌과 꿀을 테마로 한 동네 맛집 탐색 웹앱이다.  
이 문서는 Codex가 Home, Detail, Ranking, Wishlist, MyPage 화면을 리팩토링할 때 화면 구조와 우선순위를 일관되게 유지하도록 하기 위한 기준이다.

Codex는 이 문서를 기준으로 화면을 한 번에 전부 갈아엎지 않고, 작은 단위로 안전하게 리팩토링해야 한다.

---

## 2. 전체 화면 공통 기준

### 2.1 모바일 기준

Honeytong은 모바일 우선 웹앱이다.

기준 모바일 화면:

```txt
393px x 852px
```

앱 최대 폭:

```txt
430px
```

권장 기본 레이아웃:

```tsx
<div className="min-h-screen bg-neutral-100">
  <main className="mx-auto min-h-screen max-w-[430px] bg-[#fffaf0] pb-24">
    {/* page content */}
  </main>
</div>
```

---

### 2.2 화면 배경

앱 내부 배경은 따뜻한 크림색을 기본으로 한다.

```txt
#fffaf0
```

Tailwind 예시:

```tsx
bg-[#fffaf0]
```

데스크톱 바깥 배경은 연한 회색을 사용할 수 있다.

```tsx
bg-neutral-100
```

---

### 2.3 기본 페이지 여백

페이지 좌우 여백:

```tsx
px-4
```

상단 여백:

```tsx
pt-5
```

섹션 간격:

```tsx
space-y-6
```

하단 네비게이션 대응 여백:

```tsx
pb-24
```

---

### 2.4 하단 네비게이션

주요 화면에는 하단 네비게이션을 유지한다.

기본 메뉴:

```txt
홈
랭킹
찜
마이
```

주의:

- 본문이 하단 네비게이션에 가려지면 안 된다.
- 현재 경로에 맞는 메뉴가 활성화되어야 한다.
- 상세 화면에서는 기획에 따라 하단 네비게이션을 유지하거나 하단 CTA를 우선할 수 있다.
- 기존 라우팅 경로를 임의로 변경하지 않는다.

---

### 2.5 상태 처리

API 기반 화면은 다음 상태를 반드시 고려한다.

```txt
loading
success
empty
error
```

상태별 문구는 한국어를 사용한다.

예시:

```txt
꿀맛집을 불러오는 중이에요...
맛집 정보를 불러오지 못했어요.
아직 보여줄 맛집이 없어요.
```

---

### 2.6 공통 금지 사항

모든 화면에서 다음을 금지한다.

- 기존 라우팅 변경
- 기존 API 연결 변경
- mock 데이터 재도입
- 기능 삭제
- 화면 전체를 한 번에 갈아엎기
- 영어 UI 문구로 변경
- 너무 강한 노랑 배경 반복
- 과한 그림자
- 하단 네비게이션이 본문을 가리는 상태 방치
- TypeScript 오류가 있는 상태로 작업 종료

## 3. Home 화면 명세

### 3.1 화면 목적

Home 화면은 Honeytong의 첫인상을 결정하는 메인 화면이다.

사용자는 Home 화면에서 다음을 바로 이해해야 한다.

```txt
우리 동네 맛집을 탐색할 수 있다.
동네 사람들이 추천한 맛집을 볼 수 있다.
마음에 드는 맛집을 찜할 수 있다.
맛집 카드를 눌러 상세 정보를 볼 수 있다.
```

---

### 3.2 Home 정보 구조

Home 화면은 다음 순서를 권장한다.

1. 상단 브랜드/인사 영역
2. 검색창 또는 지역 안내 영역
3. 카테고리 칩
4. 추천 맛집 섹션
5. 인기 상승 맛집 또는 랭킹 미리보기
6. 하단 네비게이션

---

### 3.3 Home 상단 영역

#### 목적

Honeytong의 브랜드 감성과 사용자의 현재 목적을 전달한다.

#### 권장 문구

```txt
오늘은 어디서 꿀맛을 찾을까요?
우리 동네 사람들이 추천한 맛집을 모아봤어요.
```

또는:

```txt
우리 동네 꿀맛집을 찾아볼까요?
동네 사람들이 자주 찾는 맛집을 소개해드려요.
```

#### 디자인 기준

- 배경은 크림색 유지
- 제목은 `text-xl` 또는 `text-2xl`
- 설명은 `text-sm`
- 작은 벌/꿀 아이콘은 사용 가능
- 너무 큰 캐릭터 이미지는 피한다.

---

### 3.4 Home 검색 영역

#### 목적

사용자가 맛집, 메뉴, 지역을 검색할 수 있다는 진입점을 제공한다.

#### Placeholder

```txt
동네 맛집이나 메뉴를 검색해보세요
```

#### 디자인 기준

```tsx
className="flex h-12 items-center gap-2 rounded-2xl border border-gray-200 bg-white px-4 text-sm shadow-sm"
```

#### 주의

검색 기능이 아직 구현되지 않았다면 다음 중 하나로 처리한다.

- 비활성 스타일 적용
- 클릭 시 준비 중 안내
- 검색 기능을 실제처럼 보이게 만들지 않기

준비 중 문구:

```txt
검색 기능은 곧 추가될 예정이에요.
```

---

### 3.5 Home 카테고리 칩

#### 목적

사용자가 맛집 카테고리를 빠르게 선택할 수 있게 한다.

#### 기본 카테고리 예시

```txt
전체
한식
분식
카페
디저트
중식
일식
양식
혼밥
가성비
```

#### 디자인 기준

선택 전:

```tsx
className="rounded-full border border-gray-200 bg-white px-4 py-2 text-sm font-medium text-gray-600"
```

선택 후:

```tsx
className="rounded-full bg-[#f6b800] px-4 py-2 text-sm font-semibold text-[#2b210f]"
```

#### UX 기준

- 가로 스크롤을 허용한다.
- 선택된 칩은 명확히 보인다.
- 터치 영역은 충분히 크다.
- 선택 후 결과가 없으면 EmptyState를 보여준다.

---

### 3.6 Home 추천 맛집 섹션

#### 목적

사용자에게 가장 먼저 볼 만한 맛집을 보여준다.

#### 섹션 제목 예시

```txt
오늘의 꿀맛집
우리 동네 추천 맛집
동네 사람들이 자주 찾는 곳
```

#### 카드에 포함할 정보

- 대표 이미지
- 맛집 이름
- 지역
- 거리
- 카테고리
- 한 줄 설명
- 추천 수 또는 꿀 점수
- 찜 상태

#### 카드 동작

- 카드 전체 클릭 시 Detail 화면으로 이동
- 찜 버튼 클릭 시 찜 상태만 변경
- 찜 버튼 클릭 시 Detail 이동이 발생하면 안 됨

---

### 3.7 Home 인기 상승 / 랭킹 미리보기

#### 목적

Ranking 화면으로 자연스럽게 이동할 수 있는 관심 요소를 제공한다.

#### 섹션 제목 예시

```txt
요즘 뜨는 꿀맛집
우리 동네 인기 상승 맛집
```

#### 주의

- MVP에서 데이터가 부족하면 이 섹션은 작게 보여주거나 생략할 수 있다.
- 기능이 구현되지 않은 랭킹 버튼을 활성 버튼처럼 만들지 않는다.
- Ranking 화면이 구현되어 있다면 “전체 보기”로 연결한다.

---

### 3.8 Home 상태별 화면

#### Loading

```txt
꿀맛집을 불러오는 중이에요...
```

권장:

- SkeletonCard 3~5개
- 짧은 로딩 문구

#### Empty

```txt
아직 보여줄 맛집이 없어요.
조금 뒤에 다시 확인해보세요.
```

CTA 가능:

```txt
다시 불러오기
```

#### Error

```txt
맛집 정보를 불러오지 못했어요.
잠시 후 다시 시도해주세요.
```

버튼:

```txt
다시 불러오기
```

## 4. Detail 화면 명세

### 4.1 화면 목적

Detail 화면은 사용자가 특정 맛집에 대해 더 자세히 확인하고, 찜하거나 추천할지 결정하는 화면이다.

사용자는 Detail 화면에서 다음을 확인할 수 있어야 한다.

```txt
맛집 이름
대표 이미지
지역
거리
카테고리
추천 수 또는 꿀 점수
상세 설명
운영 정보
위치 정보
찜 상태
```

---

### 4.2 Detail 정보 구조

Detail 화면은 다음 순서를 권장한다.

1. 상단 대표 이미지
2. 뒤로가기 버튼
3. 찜 버튼
4. 맛집 이름
5. 지역 / 거리 / 카테고리
6. 추천 수 / 꿀 점수
7. 상세 설명
8. 운영 정보
9. 위치 정보
10. 추천하기 / 방문 인증 / 길찾기 CTA

---

### 4.3 Detail 상단 이미지

#### 디자인 기준

```tsx
className="aspect-[16/10] w-full object-cover"
```

또는 하단 둥근 처리:

```tsx
className="aspect-[16/10] w-full rounded-b-3xl object-cover"
```

#### 규칙

- 대표 이미지는 화면 상단에 크게 배치한다.
- 이미지가 없으면 fallback UI를 제공한다.
- 뒤로가기와 찜 버튼은 이미지 위에 둘 수 있다.
- 이미지 위 버튼은 충분히 잘 보여야 한다.
- 이미지 위에 텍스트를 과하게 올리지 않는다.

---

### 4.4 Detail 상단 버튼

#### 뒤로가기 버튼

- 항상 제공한다.
- 누르면 이전 화면 또는 Home으로 이동한다.
- 충분한 터치 영역을 가진다.

권장 스타일:

```tsx
className="flex h-11 w-11 items-center justify-center rounded-full bg-white/90 shadow-sm"
```

#### 찜 버튼

- 현재 찜 상태를 보여준다.
- 클릭 시 찜 상태만 바뀐다.
- 다른 클릭 이벤트와 충돌하지 않는다.

---

### 4.5 Detail 핵심 정보 영역

#### 예시 문구

```txt
성수 꿀분식
성수동 · 분식 · 650m
동네 사람들이 자주 추천한 꿀맛집이에요.
```

#### 디자인 기준

- 맛집 이름은 `text-2xl font-bold`
- 메타 정보는 `text-sm text-gray-500`
- 설명은 `text-sm text-gray-700`
- 핵심 정보 영역은 카드 또는 독립 섹션으로 구성한다.

---

### 4.6 Detail 정보 섹션

상세 정보는 섹션별로 구분한다.

권장 섹션:

```txt
이 맛집은요
운영 정보
위치 정보
추천 이유
방문 인증
```

InfoSection 권장 스타일:

```tsx
className="rounded-3xl bg-white p-4 shadow-sm"
```

없는 데이터 fallback 예시:

```txt
아직 상세 정보가 준비되지 않았어요.
```

---

### 4.7 Detail CTA

#### 가능한 CTA

```txt
찜하기
추천하기
방문 인증하기
길찾기
```

#### 규칙

- 실제 구현된 기능만 활성화한다.
- 미구현 기능은 비활성화하거나 준비 중 안내를 보여준다.
- CTA 버튼은 하단 근처 또는 고정 영역에 둘 수 있다.
- 하단 네비게이션과 CTA가 겹치지 않게 한다.

미구현 안내:

```txt
곧 추가될 기능이에요.
```

---

### 4.8 Detail 상태별 화면

#### Loading


#### Error

## 5. Ranking 화면 명세

### 5.1 화면 목적

Ranking 화면은 Honeytong의 차별성을 보여주는 화면이다.

사용자는 Ranking 화면에서 동네, 구, 시 단위로 인기 있는 맛집을 확인할 수 있어야 한다.

---

### 5.2 Ranking 정보 구조

Ranking 화면은 다음 순서를 권장한다.

1. 화면 제목
2. 랭킹 설명 문구
3. 지역 단위 필터
4. Top 3 강조 영역
5. 전체 랭킹 리스트
6. 하단 네비게이션

---

### 5.3 Ranking 상단 문구

권장 제목:

```txt
우리 동네 꿀맛집 랭킹
```

설명 문구:

```txt
동네 사람들의 추천과 활동을 바탕으로 정리했어요.
```

또는:

```txt
최근 추천, 찜, 방문 활동을 바탕으로 인기 맛집을 보여드려요.
```

---

### 5.4 Ranking 필터

#### 필터 예시

```txt
동네
구
시
```

#### 디자인 기준

선택 전:

```tsx
className="rounded-full bg-white px-4 py-2 text-sm font-medium text-gray-500"
```

선택 후:

```tsx
className="rounded-full bg-[#f6b800] px-4 py-2 text-sm font-semibold text-[#2b210f]"
```

#### 규칙

- 선택된 필터는 명확히 보인다.
- 필터 전환 시 목록이 변경된다.
- 구현되지 않은 필터는 활성화하지 않는다.
- 필터가 미구현이면 준비 중 안내를 보여준다.

---

### 5.5 Ranking Top 3 영역

#### 목적

상위 3개 맛집을 조금 더 눈에 띄게 보여준다.

#### 표시 정보

- 순위
- 맛집 이름
- 대표 이미지
- 지역
- 추천 수
- 짧은 이유

#### 디자인 기준

- 1~3위는 약간 더 강조한다.
- 꿀색 배경이나 배지를 사용할 수 있다.

```tsx
className="rounded-3xl bg-white p-4 shadow-sm"
```
